# Use grilling as the only Matt requirements interrogator

## 问题

- 需求拷问应由上游 ask-matt 完整流程还是窄能力 grilling 承担？

## 选择

- 不启用 ask-matt 作为路由器；仅在用户明确要求逐问确认时调用 grilling，生命周期仍由 Direct、Summer 或 GSD 单独拥有。

## 拒绝

- 默认启用上游 ask-matt idea-to-ship 流程
- 继续维护名为 ask-matt 的本地影子路由器

## 为什么不选

- 两种 ask-matt 都会增加第二路由面或造成名称/语义混淆；grilling 只问决策问题，不创建并列状态。
