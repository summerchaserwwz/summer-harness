# 构建并开源 Summer Harness

## 目标

- 把 Summer Harness 迭代为易安装、易使用、默认轻量、支持可靠跨 session、多 Agent 协作、候选式自我进化和真实 GUI 看板的开源 Coding Agent Harness，并发布到 summerchaserwwz/summer-harness。

## 验收条件

- 普通任务保持 Direct-first，完整 Harness 只能显式启用，且冷恢复性能不明显退化。
- 唯一 HANDOFF 能跨独立 session 可靠恢复，权威状态与投影漂移必须 fail-closed。
- 多 Agent 工作具备任务所有权、隔离、provenance 和安全合并约束，不产生并列账本。
- 自我进化只生成有证据的候选规则，必须经用户确认后才能生效。
- GUI 提供真实任务、证据、关系、Agent、经验候选和健康状态看板，数据库或缓存只作为可重建投影。
- 提供清晰的一键安装、中文文档、示例、自动测试和公开 GitHub 仓库。

## 已完成

- M0/M1-A 已完成架构冻结、公开仓库和 Go transaction Ledger foundation。
- M1-B 已完成 Continuity vertical slice：真实 summer resume/doctor、v1 三模式恢复、v2 有界 Handoff/Capsule、缺失投影重建、生命周期/容量 fail-closed 与 CLI 边界防护。

## 下一步

- 完成迁移 fault tests、CLI 文档与独立安全审查后提交并推送。

## 验证

- M1-C 当前验证：Go migration/rollback/full-history/concurrency 集成测试与 Python 25 项回归通过。
